# Post-MVP Feature Ideas

This document tracks feature ideas and enhancements for the Hazards app that are candidates for development after the MVP launch. These are not committed features, but represent areas of interest, user requests, or potential improvements for future planning.

---

## Feature Ideas

### 1. Weather Integration
- Show current weather and forecasts for hazard/map locations.
- Optionally display weather alerts or warnings.
- Potential sources: NOAA, OpenWeatherMap, etc.

### 2. Police/Scanner Feed Integration
- Surface relevant police scanner or emergency feeds for the map area.
- Allow users to listen to live feeds or see recent incident summaries.

### 3. Owner-Only Pending Hazards
- Allow hazard creators to see their own pending/unapproved hazards on the map.
- Optionally allow editing or withdrawing pending hazards.

### 4. Map Filtering & Layer Controls
- Filter hazards by type, severity, recency, or trust score.
- Toggle map layers (e.g., satellite, terrain, traffic, weather overlays).

### 5. Offline/Trip Mode Enhancements
- Improved offline map downloads and trip planning.
- Allow users to save custom map views and hazard sets for trips.
- Sync trip data when back online.

### 6. Community & Social Features
- User profiles, badges, and reputation.
- Commenting or discussion on hazards.
- Following users or locations.

### 7. Notification Improvements
- Push notifications for new hazards, updates, or area-specific alerts.
- Customizable notification preferences.


### 9. Add Route to Map
- Allow users to create and display custom routes on the map (e.g., for planned trips, detours, or safe paths).
- Support drawing, editing, and saving routes.
- Optionally associate hazards, notes, or waypoints with a route.

### 8. Accessibility & UX
- Enhanced accessibility for screen readers and keyboard navigation.
- More robust mobile and offline UX.

---

## Tracking & Next Steps
- This list is for ideation and planning only. Features may be added, removed, or reprioritized based on user feedback and project needs.
- For more details or to propose new ideas, contact the project team or open a discussion in the project repository.
